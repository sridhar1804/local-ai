## Decisions Made

- **`model_memory/` vs `.claude/memory/` split is permanent.** `model_memory/` is what the *model* writes (traces of its own runs); `.claude/memory/` is what the *agent assistant* reads and writes (durable working knowledge across sessions). They never mix.
- **No `patterns/` directory under `.claude/memory/`.** User vetoed it. Per-component conventions stay implicit — they live in the code itself plus the global `code_style.md`.
- **Sessions are append-only.** Each session writes one new dated file. Corrections to past sessions live in the current session's file, not by editing history.
- **`v1.md` historical refs preserved.** The v1 spec describes the original `memory/` package layout; rewriting the body would falsify the historical record. The path-note banner at the top is the redirect.
- **`.claude/settings.local.json` is gitignored**, `.claude/settings.json` is committed. Standard Claude Code project convention.
